## Введение

Цель работы: реализовать на C++ систему управления базами данных с файловым хранением, SQL-подобным языком запросов и индексированием уникальных полей с помощью B+-tree.

## Основная часть

### Архитектура

Система состоит из одного исполняемого файла `cwdb`, поддерживающего интерактивный и пакетный режим. Данные размещаются в каталоге `data`: отдельная папка соответствует базе данных, отдельная папка внутри базы соответствует таблице. Схема хранится в `schema.txt`, строки таблицы в `rows.tsv`, журнал изменений в `history.log`.

### Типы данных и ограничения

Поддерживаются типы `int` и `string`, а также значение `NULL`. Модификатор `NOT_NULL` запрещает пустое значение. Модификатор `INDEXED` также запрещает `NULL`, требует уникальность и создает B+-tree индекс. Модификатор `DEFAULT value` подставляет значение при пропуске столбца в `INSERT`.

### Индекс B+-tree

Для каждого индексированного столбца создается экземпляр шаблонного класса `BPlusTree<Value, size_t>`. Ключом является значение столбца, значением — номер строки в файле таблицы. Запись целиком в индексе не дублируется, что соответствует требованию хранения индексов как ссылок на данные.

### Парсер запросов

Команды завершаются символом `;`. Реализованы `CREATE DATABASE`, `DROP DATABASE`, `USE`, `CREATE TABLE`, `DROP TABLE`, `INSERT`, `UPDATE`, `DELETE`, `SELECT`, `REVERT`. Ключевые слова распознаются регистронезависимо. Условия `WHERE` поддерживают сравнения, `BETWEEN`, `LIKE`, составные выражения `AND`, `OR` и скобки.

### Персистентность и REVERT

Перед изменением таблицы в `history.log` записывается состояние строк с временной меткой. Команда `REVERT table yyyy.mm.dd-hh:mm:ss.msmsms` восстанавливает ближайшее состояние, существовавшее не позже указанного времени. Реализация не копирует файлы базы целиком.

### Вывод

Результат `SELECT` выводится как JSON-массив объектов. Ошибки возвращаются как информативные сообщения, аварийное завершение для пользовательских ошибок предотвращается обработкой исключений.

## Вывод

В результате реализована файловая СУБД для варианта 2 с B+-tree индексами, типизированными таблицами, ограничениями целостности, SQL-подобным интерфейсом, пакетным режимом и пояснительной структурой.
